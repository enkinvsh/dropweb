# TLS Fragment (фрагментация TLS)

TLS Fragment — опциональная функция обхода DPI-блокировок. Дробит TLS ClientHello на несколько TCP-сегментов так, чтобы имя хоста (SNI) не передавалось одним куском — DPI-системы, блокирующие по SNI, не могут его прочитать и заблокировать соединение.

## Как работает

- SNI-таргетированная фрагментация: парсит ClientHello, находит позицию SNI (имени хоста) в записи и режет точно через него — изолирует часть до/после и дробит само имя хоста на куски размером не больше заданного (гарантированно ≥2 фрагмента).
- Если SNI распарсить не удалось — запасной вариант: разбивка на 3 части.
- Не-TLS первый пакет проходит без изменений.
- Реализация в Go-ядре (`component/dialer/tlsfragment.go`), применяется на этапе установки соединения (dialContext), работает и с TFO, и в обычном режиме.

## Параметры (general config)

| Параметр | Назначение | По умолчанию |
|---|---|---|
| `tls-fragment` | Включить/выключить фрагментацию | выкл (opt-in) |
| `tls-fragment-size` | Максимальный размер фрагмента (байт) | 8 |
| `tls-fragment-delay` | Задержка между фрагментами | 5ms |

## Как включить

- В приложении: тумблер «TLS Fragment» в настройках (opt-in, по умолчанию выключен).
- Через конфиг: секция general, поля выше.
- При активации в лог пишется `[TLSFragment] active`.

## Когда использовать

Включать, если провайдер блокирует соединения по SNI (DPI). Небольшой оверхед на установку соединения — поэтому opt-in, а не всегда.
