# Заголовки подписки dropweb

dropweb-app читает HTTP-заголовки ответа подписки и применяет их к интерфейсу и настройкам клиента — концепция «клиент-губка». Заголовки отправляются панелью (Remnawave) вместе с содержимым подписки. Все кастомные заголовки начинаются с `dropweb-`.

Документ описывает **темизацию** и **dropweb-disconeko**. Прочие `dropweb-*` заголовки (`servicename`, `servicelogo`, `widgets`, `view`, `globalmode`, `settings`, `cabinet`) существуют, но здесь не специфицируются.

---

## 1. dropweb-custom — гейт применения

Управляет тем, **когда** применять все `dropweb-*` настройки:

| Значение | Поведение |
|---|---|
| `add` | Применить только при первом добавлении профиля |
| `update` | Применять при каждом обновлении подписки и при переключении профиля |
| (нет заголовка / иное) | Не применять `dropweb-*` вообще |

> Без `dropweb-custom: update` тема и настройки не будут обновляться на каждом цикле. Тема применяется при **обновлении подписки** и при **переключении профиля**.

## 2. dropweb-theme — единый контракт темы (рекомендуемый)

```
dropweb-theme: <filter>,<accentHex>,<orb1Hex>,<orb2Hex>,<blur>
```

Все поля после `filter` позиционно опциональны — пропущенный или невалидный сегмент оставляет соответствующее поле без изменений.

### filter — алгоритм цветовой схемы
HSL-модификатор акцента и орбов (оттенок/hue сохраняется). Имя = как слать в заголовке; в скобках — название в интерфейсе приложения:

| filter | UI | Эффект |
|---|---|---|
| `fidelity` | Обычные | Точный акцент, без искажения. **Рекомендуемый / дефолт.** |
| `vibrant` | Яркие | Повышенная насыщенность |
| `monochrome` | Моно | Обесцвечивание (серый) |
| `neutral` | Нейтральные | Приглушённая насыщенность |
| `expressive` | Выразительные | Сдвиг оттенка |

### Остальные поля
| Поле | Назначение | Формат |
|---|---|---|
| `accentHex` | Акцент (кнопка, акценты) → primaryColor | 6 или 8 hex-цифр, `#` опционален |
| `orb1Hex` | Цвет верхних орбов фона → orbColorPrimary | hex |
| `orb2Hex` | Цвет нижнего орба фона → orbColorSecondary | hex |
| `blur` | Мягкость орбов: 1 = чёткие, 5 = диффузное свечение | число 1–5 (клампится) |

### Пример
```
dropweb-custom: update
dropweb-theme: fidelity,#15803D,#009938,#009938,2
```
Фирменный зелёный dropweb, точный фильтр, блюр 2.

> Если `orb1`/`orb2` пусты — орбы следуют за акцентом (Авто). Фильтр применяется и к акценту, и к орбам.

## 3. Пользовательский тумблер «Тема из подписки»

В приложении (**Настройки → Тема**) есть тумблер **«Тема из подписки»** (по умолчанию ВКЛ) — мастер-выключатель пользователя:

- **ВКЛ** — операторская тема (`dropweb-theme`) применяется
- **ВЫКЛ** — операторская тема игнорируется, пользователь оставляет свою

Оператор не может переопределить этот тумблер. **Для применения темы нужны оба условия:** `dropweb-custom: update` (оператор) И тумблер ВКЛ (пользователь).

## 4. dropweb-disconeko — аварийный пул серверов (SOS fallback)

```
dropweb-disconeko: <URL>
```

URL указывает на **отдельный** пул аварийных серверов (raw share-ссылки или Mihomo YAML). При обновлении подписки клиент:

1. Загружает конфиг по этому URL,
2. Парсит из него прокси,
3. Вмёрживает их в доставленный конфиг как fallback-пул (Smart-группа),
4. Валидирует результат встроенным ядром; если ядро отклоняет — оставляет непатченный конфиг.

**Назначение:** трафик автоматически переключается на аварийные сервера, когда все обычные мертвы.

> **Строго best-effort.** Пул опционален; любая ошибка (загрузка, декодирование, парсинг, валидация) тихо пропускается и **никогда** не ломает основное обновление подписки.

### Пример
```
dropweb-disconeko: https://example.com/emergency-pool
```

## 5. dropweb-cabinet — действие личного кабинета

```
dropweb-cabinet: <HTTPS URL>
```

URL указывает на личный кабинет пользователя. Когда заголовок присутствует, приложение показывает действие «личный кабинет» и открывает указанный URL в браузере или custom tab.

**Требования:**

- значение должно быть полным HTTPS URL (например `https://cab.example.com`);
- если заголовка нет, приложение не показывает действие личного кабинета.

### Пример
```
dropweb-cabinet: https://cab.example.com
```

## 6. dropweb-logo — логотип провайдера на карточке

```
dropweb-logo: <URL картинки>
```

URL указывает на логотип провайдера (PNG/JPG/SVG; допускается base64-строка). Когда заголовок присутствует, приложение показывает **круглый логотип** в шапке карточки подписки (на месте бывшей иконки меню — само меню теперь открывается свайпом снизу-вверх по дашборду).

- Логотип обрезается в круг и обводится тонкой **акцентной обводкой**, которая загорается вместе с кнопкой подключения: яркий акцент при подключённом VPN, приглушённый — при отключённом.
- К логотипу применяется активный **фильтр темы** (`fidelity`/`vibrant`/`monochrome`/…), как к акценту и орбам — например при «Моно» логотип становится чёрно-белым.
- Если заголовка нет — логотип не показывается.

### Пользовательский тумблер «Лого из подписки»

В приложении (**Настройки → Тема**, над «Тема из подписки») есть тумблер **«Лого из подписки»** (по умолчанию ВКЛ). ВЫКЛ — логотип скрывается, даже если панель его отдаёт.

### Пример
```
dropweb-logo: https://example.com/logo.png
```

## 7. dropweb-renew-url / dropweb-topup-url — кнопки монетизации

Две ссылки, по которым на карточке подписки появляются **контекстные** кнопки. Логика показа — на стороне клиента; сами ссылки задаёт панель (могут быть персональными, с токеном пользователя). Кнопки показываются только когда это уместно — чтобы не быть постоянным баннером.

```
dropweb-renew-url: <HTTPS URL оплаты/продления>
dropweb-topup-url: <HTTPS URL докупки трафика>
```

| Заголовок | Кнопка | Когда показывается |
|---|---|---|
| `dropweb-renew-url` | «Продлить» | до конца подписки осталось меньше **3 дней** (в т.ч. если уже истекла) |
| `dropweb-topup-url` | «Докупить трафик» | остаётся меньше **10%** от лимита трафика |

- Условия показа считаются из стандартного `subscription-userinfo` (`expire`, `total`, `upload`+`download`). Без этих данных кнопки не появляются.
- `dropweb-renew-url` требует план с датой окончания (`expire > 0`), `dropweb-topup-url` — план с лимитом трафика (`total > 0`).
- Сработали **оба** триггера — кнопки в один ряд; один — кнопка на всю ширину.
- По нажатию открывается переданный URL (с подтверждением). Нет заголовка или условие не выполнено — кнопка не показывается.

### Пример
```
dropweb-renew-url: https://panel.example.com/pay
dropweb-topup-url: https://panel.example.com/buy-traffic
```
